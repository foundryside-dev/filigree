## 1. Why a major

2.x deferred a cluster of **breaking wire-surface changes** because shipping them
mid-2.x would break federation consumers (Clarion / Wardline / Shuttle) without a
coordinated migration window. 3.0.0 opens that boundary. The release **bundles**
those breaking items (per the maintainer decision "bundle breaking work first"),
so the PR is not release-ready until the breaking checklist in §4 is complete.

(`9b4bb6e52e`) → safe_message (`d25e75cebf`) → MCP namespacing (`7771610917`,
